## Schema

### Required

- `cluster_id` (String) ID of the Kubernetes Cluster.

### Optional

- `scale_down_delay_after_add` (String) How long after scale up that scale down evaluation resumes. Unit: seconds. Default is 300.
- `scale_down_delay_after_delete` (String) How long after node deletion that scale down evaluation resumes. Unit: seconds. Default is 2.
- `scale_down_delay_after_failure` (String) How long after scale down failure that scale down evaluation resumes. Unit: seconds. Default is 120.
- `scale_down_unneeded_time` (String) How long a node should be unneeded before it is eligible for scale down. Unit: seconds. Default is 120.
- `scale_down_utilization_threshold` (String) Node utilization level, defined as sum of requested resources divided by capacity, below which a node can be considered for scale down. Range: 0.0 to 1.0. Default is 0.1.
- `scan_interval` (String) How often cluster is reevaluated for scale up or down. Unit: seconds. Default is 10.

### Read-Only

- `id` (String) The ID of the config, same as cluster_id.
